To see how CoralNet is doing overall with processing jobs, check out the sitewide status page. (There is also a link to this page on the site's footer.)

Types of jobs:

  • Extract features: A pre-processing step that must be run for every image, before that image can be classified or used for training a classifier.
  • Train classifier: Trains a new classifier for the source, using all the confirmed and feature-extracted images in the source up to that point. The new classifier may or may not be saved, depending on how the evaluation results compare to the source's previous classifier.
  • Classify: Classifies an image, using the source's selected classifier.
  • Check source: Checks whether it's time to schedule more feature-extract, training, or classification jobs for the source. The general strategy this job uses is:
    • First extract features for all images.
    • Then check if the criteria are met for a new training job.
    • Then make sure all non-confirmed images are classified with the selected classifier.
  • Reset classifiers for source: Clear all previous saved classifiers for the source. Certain changes, such as changing the source's labelset, will result in this job being scheduled.
  • Reset features for source: Clear all previous saved extracted features for the source. Changing the source's feature-extractor setting results in this job being scheduled (unless no feature extractions have been done yet).
