Your CSV (comma-separated values) file should have one row per point. The first row must contain column headers that specify which column is which. Here's an example:

Name Row Column Label code
IMG_0056.jpg 1528 426 Porit
IMG_0056.jpg 786 1104 CCA
IMG_0056.jpg 39 1576 Off
DSC_0162__postprocess.png 2108 960 Favia
DSC_0162__postprocess.png 1772 1435 Turf
  • Name: The name of the image that the point is for.
  • Row: Pixel y-position from the top edge of the image.
  • Column: Pixel x-position from the left edge of the image.
  • Label code or Label ID: Label to annotate the point with, if any.

Labels can be specified using the labelset's short codes (in the Label code case), or using site-wide label ID numbers (Label ID). These labels will be applied as confirmed annotations. If no label is specified for a particular point, then CoralNet will not apply a confirmed annotation to that point.

You can use a spreadsheet program such as Excel to create the CSV file. It's not much different from creating a typical Excel spreadsheet; just select the CSV format in Excel's "Save As" dialog. Alternatively, depending on how your point and annotation data is organized, you could automate the CSV file creation with a programming language like Python.

Details on how your CSV file will be processed:

  • If you specify any points for an image, all existing points and annotations for that image will be overwritten. The preview will warn you if you are overwriting any existing confirmed annotations.
  • Name, Row, and Column must be specified in every CSV row. Label code or Label ID can be specified in all, any, or none of the CSV rows.
  • For backward compatibility with older CoralNet exports, the column header Label is also accepted, and interpreted the same as Label code.
  • If your image is 1000 pixels wide, then the possible column numbers are 0 to 999. The same thing applies to row numbers and image height.
  • Label codes or IDs in the CSV file must correspond to a label in your source's labelset.
  • Label codes are case-insensitive. If your labelset has a label code PORIT, and the CSV file has Porit, then these codes will match.
  • You may include CSV rows with image names that aren't in your source. Those rows will just be ignored.
  • You may include extra CSV columns with names outside of the recognized column names. Those columns will just be ignored.
